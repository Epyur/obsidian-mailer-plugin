| ## Возможности | ## Features |
| **Управление письмами**: создание, редактирование, удаление писем с привязкой к направлениям.<br><br>**Изображения в письмах**: загрузка PNG/JPEG/BMP/GIF/TIFF, вставка плейсхолдеров `{IMG_N}`, отображение в MD.<br><br>**Поиск и группировка**: поиск по тексту/теме/номеру, группировка по направлениям с коллапсом.<br><br>**Направления**: управление справочником направлений деятельности.<br><br>**Импорт из JSON**: импорт сторонних баз писем с маппингом направлений (создать/сопоставить/пропустить).<br><br>**Экспорт в JSON**: экспорт писем по одному или нескольким направлениям в JSON-файл (`Технические письма/Экспорт/`).<br><br>**Экспорт в Word**: генерация DOCX с поддержкой кастомных шаблонов (JSZip) и плейсхолдеров `{{Номер}}`, `{{Тема}}`, `{{Текст}}`, `{{Автор}}`, `{{Дата}}`. Без шаблона — fallback через `docx` библиотеку.<br><br>**Синхронизация с MD**: каждое письмо синхронизируется в `.md` файл в папке `Технические письма/` (замена `{IMG_N}` на `![[path]]`).<br><br>**Облачная синхронизация**: push/pull с REST API (опционально).<br><br>**AI-ассистент**: чат с OpenAI-совместимым LLM, RAG по базе писем, анализ документов.<br><br>**Локальное хранилище**: все данные в `mailer_data.json` с авто-бэкапом каждые 5 сохранений. | **Email management**: create, edit, delete emails linked to activity directions.<br><br>**Images in emails**: upload PNG/JPEG/BMP/GIF/TIFF, insert `{IMG_N}` placeholders, render in MD.<br><br>**Search & grouping**: search by text/subject/number, collapsible grouping by directions.<br><br>**Directions**: manage the directory of activity directions.<br><br>**JSON import**: import third-party email databases with direction mapping (create/map/skip).<br><br>**JSON export**: export emails by one or multiple directions to a JSON file (`Технические письма/Экспорт/`).<br><br>**Word export**: DOCX generation with custom template support (JSZip) and placeholders `{{Номер}}`, `{{Тема}}`, `{{Текст}}`, `{{Автор}}`, `{{Дата}}`. Without template — fallback via `docx` library.<br><br>**MD sync**: each email synced to a `.md` file in the `Технические письма/` folder (replacing `{IMG_N}` with `![[path]]`).<br><br>**Cloud sync**: push/pull with REST API (optional).<br><br>**AI assistant**: chat with OpenAI-compatible LLM, RAG over email database, document analysis.<br><br>**Local storage**: all data in `mailer_data.json` with auto-backup every 5 saves. |

| ## Работа с изображениями | ## Working with images |
| 1. При создании/редактировании письма нажмите "📎 Добавить изображение"<br>2. Выберите файл (PNG, JPEG, BMP, GIF, TIFF)<br>3. Плейсхолдер `{IMG_N}` автоматически вставляется в текст в позиции курсора<br>4. Файл копируется в `Технические письма/Изображения/`<br>5. В `.md` файле плейсхолдер заменяется на `![[path]]` (Obsidian wikilink)<br>6. В `.docx` — плейсхолдеры `{IMG_N}` остаются как текст (экспорт изображений в DOCX в разработке) | 1. When creating/editing an email, click "📎 Добавить изображение"<br>2. Select a file (PNG, JPEG, BMP, GIF, TIFF)<br>3. The `{IMG_N}` placeholder is inserted at the cursor position<br>4. File is copied to `Технические письма/Изображения/`<br>5. In `.md` files the placeholder is replaced with `![[path]]` (Obsidian wikilink)<br>6. In `.docx` — `{IMG_N}` placeholders remain as text (DOCX image export in development) |
| ## Экспорт/импорт JSON | ## JSON Export/Import |

Нажмите "📥 Импорт" в панели писем<br>2. Выберите JSON-файл с массивами `emails` и `directions`<br>3. Настройте соответствие направлений: создать новое, сопоставить с существующим или пропустить<br>4. Подтвердите импорт | **Export**:<br>1. Click "📤 Экспорт JSON" in the email panel<br>2. Select one or multiple directions<br>3. File is saved to `Технические письма/Экспорт/`<br><br>**Import**:<br>1. Click "📥 Импорт" in the email panel<br>2. Select a JSON file with `emails` and `directions` arrays<br>3. Configure direction mapping: create new, map to existing, or skip<br>4. Confirm the import |
