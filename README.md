# Technical assistence in mail writing

| 🇷🇺 Русский | 🇬🇧 English |
|---|---|
| **Технический помощник TECHNONICOL** — плагин для Obsidian, предназначенный для ведения базы технических писем по строительным материалам и системам TECHNONICOL. | **Technical Assistant TECHNONICOL** is an Obsidian plugin for managing a database of technical correspondence on TECHNONICOL construction materials and systems. |
| ## Возможности | ## Features |
| — **Управление письмами**: создание, редактирование, удаление писем с привязкой к направлениям<br>— **Изображения в письмах**: загрузка PNG/JPEG/BMP/GIF/TIFF, вставка плейсхолдеров `{IMG_N}`, отображение в MD<br>— **Поиск и группировка**: поиск по тексту/теме/номеру, группировка по направлениям с коллапсом<br>— **Направления**: управление справочником направлений деятельности<br>— **Импорт из JSON**: импорт сторонних баз писем с маппингом направлений<br>— **Экспорт в Word**: генерация DOCX с поддержкой кастомных шаблонов и плейсхолдеров `{{Номер}}`, `{{Тема}}`, `{{Текст}}`, `{{Автор}}`, `{{Дата}}`<br>— **Синхронизация с MD**: каждое письмо синхронизируется в `.md` файл в папке `Технические письма/`<br>— **Облачная синхронизация**: push/pull с REST API (опционально)<br>— **AI-ассистент**: чат с OpenAI-совместимым LLM, RAG по базе писем, анализ документов<br>— **Локальное хранилище**: все данные в `mailer_data.json` с авто-бэкапом | — **Email management**: create, edit, delete emails linked to activity directions<br>— **Images in emails**: upload PNG/JPEG/BMP/GIF/TIFF, insert `{IMG_N}` placeholders, render in MD<br>— **Search & grouping**: search by text/subject/number, collapsible grouping by directions<br>— **Directions**: manage the directory of activity directions<br>— **JSON import**: import third-party email databases with direction mapping<br>— **Word export**: DOCX generation with custom template support and placeholders `{{Номер}}`, `{{Тема}}`, `{{Текст}}`, `{{Автор}}`, `{{Дата}}`<br>— **MD sync**: each email synced to a `.md` file in the `Технические письма/` folder<br>— **Cloud sync**: push/pull with REST API (optional)<br>— **AI assistant**: chat with OpenAI-compatible LLM, RAG over email database, document analysis<br>— **Local storage**: all data in `mailer_data.json` with auto-backup |
| ## Установка | ## Installation |
| 1. Скопируйте папку плагина в `.obsidian/plugins/obsidian-mailer-plugin/` вашего хранилища<br>2. В Obsidian: `Настройки → Сторонние плагины → Включить`<br>3. Активируйте плагин "Технический помощник TECHNONICOL"<br>4. В боковой панели появится кнопка 📧 для открытия списка писем | 1. Copy the plugin folder to your vault's `.obsidian/plugins/obsidian-mailer-plugin/`<br>2. In Obsidian: `Settings → Community plugins → Enable`<br>3. Activate the "Technical Assistant TECHNONICOL" plugin<br>4. A 📧 button will appear in the sidebar to open the email list |
| ## Команды | ## Commands |
| `📧 Open emails` — открыть панель писем<br>`🔄 Sync with cloud` — синхронизация с облаком<br>`📊 Local DB status` — статус локальной базы | `📧 Open emails` — open the email panel<br>`🔄 Sync with cloud` — sync with the cloud<br>`📊 Local DB status` — local database status |
| ## Настройки | ## Settings |
| **☁️ Cloud sync**: API URL, токен доступа, кнопка синхронизации<br>**📝 Local settings**: автор по умолчанию, плейсхолдеры для шаблонов<br>**📄 Export templates**: путь к шаблону DOCX, создание/проверка шаблона<br>**🧠 AI Assistant**: URL API LLM, ключ API, модель, системный промпт | **☁️ Cloud sync**: API URL, access token, sync button<br>**📝 Local settings**: default author, template placeholders<br>**📄 Export templates**: DOCX template path, create/check template<br>**🧠 AI Assistant**: LLM API URL, API key, model, system prompt |
| ## Формат данных | ## Data Format |
| **Email**: `{id, number, subject, text, author, date, direction_id, images[], mdFilePath, mdFileHash, lastSyncTime, sync_status, created_at}`<br>**Direction**: `{id, name, description, created_at}`<br>Хранилище: `mailer_data.json`<br>Изображения: `Технические письма/Изображения/` | **Email**: `{id, number, subject, text, author, date, direction_id, images[], mdFilePath, mdFileHash, lastSyncTime, sync_status, created_at}`<br>**Direction**: `{id, name, description, created_at}`<br>Storage: `mailer_data.json`<br>Images: `Технические письма/Изображения/` |
| ## Работа с изображениями | ## Working with images |
| 1. При создании/редактировании письма нажмите "📎 Добавить изображение"<br>2. Выберите файл (PNG, JPEG, BMP, GIF, TIFF)<br>3. Плейсхолдер `{IMG_1}` автоматически вставится в текст в позиции курсора<br>4. В `.md` файле плейсхолдер заменяется на `![[path]]` (Obsidian wikilink, работает без ошибок)<br>5. В `.docx` файле плейсхолдеры `{IMG_N}` остаются как текст (экспорт изображений в DOCX в разработке) | 1. When creating/editing an email, click "📎 Добавить изображение"<br>2. Select a file (PNG, JPEG, BMP, GIF, TIFF)<br>3. The `{IMG_1}` placeholder is inserted at the cursor position<br>4. In `.md` files the placeholder is replaced with `![[path]]` (Obsidian wikilink, works correctly)<br>5. In `.docx` files `{IMG_N}` placeholders remain as text (DOCX image export in development) |
| ## Экспорт/импорт JSON | ## JSON Export/Import |
| **Экспорт**:<br>1. Нажмите "📤 Экспорт JSON" в панели писем<br>2. Выберите одно или несколько направлений<br>3. Файл сохранится в `Технические письма/Экспорт/`<br><br>**Импорт**:<br>1. Нажмите "📥 Импорт" в панели писем<br>2. Выберите JSON-файл с массивами `emails` и `directions`<br>3. Настройте соответствие направлений: создать новое, сопоставить с существующим или пропустить<br>4. Подтвердите импорт | **Export**:<br>1. Click "📤 Экспорт JSON" in the email panel<br>2. Select one or multiple directions<br>3. File is saved to `Технические письма/Экспорт/`<br><br>**Import**:<br>1. Click "📥 Импорт" in the email panel<br>2. Select a JSON file with `emails` and `directions` arrays<br>3. Configure direction mapping: create new, map to existing, or skip<br>4. Confirm the import |
